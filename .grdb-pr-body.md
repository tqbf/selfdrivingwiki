## Key findings

- **GRDB can replace all hand-rolled plumbing** while preserving the `mutate()` seam and `WikiEventBus`
- **The `mutate()` seam is portable** — it's 23 lines that work as a thin wrapper around `dbQueue.write`, with the same depth-tracking and flush-after-commit semantics
- **`ValueObservation` complements, not replaces** the event bus — the File Provider needs `(kind, id, change)` metadata, not raw query re-fetch
- **`DatabaseMigrator` eliminates the fresh-schema-vs-ladder dual path** — a persistent `grdb_migrations` table means fresh and existing DBs run the same migrations
- **Custom extensions (sqlite-vec, zstd) register via `prepareDatabase`** on every connection — cleaner than the current manual dual-init registration
- **QueueStore is the ideal pilot** — separate DB, simple schema, no event bus, 19 methods, 3-version ladder
